本栏目收录计算机网络协议相关的学习资料，包括 TCP/IP、HTTP、HTTPS 等核心协议。

📖 网络协议介绍

网络协议是计算机之间进行通信的规则和标准。理解网络协议是每个软件工程师必备的基础知识，对于开发网络应用、排查网络问题、优化系统性能都至关重要。

🎯 为什么要学习网络协议

  1. 理解网络通信：明白数据在网络中如何传输
  2. 排查网络问题：快速定位和解决网络故障
  3. 优化应用性能：了解协议特性，优化网络性能
  4. 保障系统安全：理解安全协议，防范网络攻击
  5. 技术面试必备：大厂面试常考网络协议知识

📊 OSI 七层模型与 TCP/IP 四层模型

🌐 核心网络协议

应用层协议

HTTP/HTTPS：

  • HTTP：Web 通信基础协议，无状态，基于请求-响应
  • HTTPS：HTTP + SSL/TLS 加密，数据加密传输

DNS：域名系统，将域名转换为 IP 地址

其他协议：FTP、SMTP、WebSocket、gRPC

传输层协议

TCP（传输控制协议）：

  • 面向连接、可靠传输
  • 三次握手、四次挥手
  • 适用：HTTP、FTP、SSH

UDP（用户数据报协议）：

  • 无连接、不可靠、低延迟
  • 适用：视频直播、在线游戏

网络层协议

IP（网际协议）：

  • IPv4：32 位地址（192.168.1.1）
  • IPv6：128 位地址

ICMP：网络诊断（ping 命令）

ARP：IP 地址转 MAC 地址
